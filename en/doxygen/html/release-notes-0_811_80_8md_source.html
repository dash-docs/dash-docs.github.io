1 0.11.0 Release notes
2 ====================
3 
4 Darkcoin Core 0.11.0 is forked off the Bitcoin Core 0.9.3. The old darkcoin
5 versions 0.10.x were forked from the Litecoin 0.8 tree. The following changes
6 are introduced in this major release. This list is compiled from the release
7 notes of Bitcoin Core 0.9.0, 0.9.1, 0.9.2, 0.9.2.1, 0.9.3 and was completed
8 with the introduced changes to the Darkcoin Core 0.11.0.
9 
10 
11 Darkcoin Core:
12 
13 - Rebrand to `Darkcoin Core`
14 - Version bumped to 0.11 to indicate a new major release
15 - Renamed client to identify with network from `Satoshi` to `Core`
16 - Bumped protocol version to 70052
17 - Changed p2sh-address versions to start with `7` (instead of `3`, this affects
18  public keys only, old scripts remain valid and usable!)
19 - Changed testnet address versions to start with `x` or `y` (instead of `m` or
20  `n`, this affects public keys only, old wallets remain valid and usable!)
21 - Defined BIP32 (HD) address versions to start with `drkp`/`drkv` (`DRKP`/`DRKV`
22  for testnet)
23 - Adapted BIP44 coin type `5` for Darkcoin (0x80000005) as defined in SLIP-0044
24 - Added new units: `duffs` (1 / 100.000.000 DRK)
25 - Added units for testnet: tDRK, mtDRK, utDRK, tduffs
26 - Added new DNS seed from masternode.io
27 - Fixed wallet locking after sending coins
28 - Add `-regtest` mode, similar to testnet but private with instant block
29  generation with `setgenerate` RPC.
30 - Add separate darkcoin-cli client
31 - Implemented KeyPass integration for CLI, RPC and Qt:
32  `keepass`, `keepassport`, `keepasskey`, `keepassid`, `keepassname`
