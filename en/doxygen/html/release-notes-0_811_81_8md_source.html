1 0.11.1 Release notes
2 ====================
3 
4 Darkcoin Core 0.11.1 supports a full implementation of InstantX, Darksend improvements
5 and a new version of enforcement compatible with the newer Bitcoin architechure.
6 Latest release in 0.11.1.x tree is v0.11.1.26, which is rebranding Darkcoin to Dash.
7 
8 - Fully implemented IX
9 - Added support for DSTX messages, as a result DS should be much faster
10 - Clear vValue in SelectCoinsMinConf - should fix an issue with conflicted txes
11 - "Debug window" -> "Tools window" renaming
12 - "Last Darksend message" text added in overview page
13 - Many new languages are supported, such as German, Vietnamese, Spanish
14 - Fixed required maturity of coins before sending
15 - New masternode payments enforcement implementation
16 - Added support to ignore IX confirmations when needed
17 - Added --instantxdepth, which will show X confirmations when a transaction lock is present
18 - fix coin control crash https://github.com/bitcoin/bitcoin/pull/5700
19 - always get only confirmed coins by AvailableCoins for every DS relative action
20 - New languages supported Portuguese, German, Russian, Polish, Spanish, Vietnamese, French,
21 Italian, Catalan, Chinese, Danish, Finnish, Swedish, Czech, Turkish and Bavarian (and many more)
22 - Full implementation of spork. Currently this includes 4 different sporks, InstantX, InstantX block enforcement, masternode payments enforcement, and reconverge. This uses the inventory system, so it's super efficient and very powerful for future development. Reconverge will put the blockchain into a mode where it will attempt to reprocess any forks without restarting the client, this means if we even had a fork in the future this can be triggered to fix it without any damage to the network.
23 - Masternode payments now uses the inventory system, which will result in much lower bandwidth usage
24 - Improved caching
25 - use wallet db ds flag in rpc
26 - update tx types in UI / fix tx decomposition / use wallet db ds flag
27 - use wallet db to save ds flag / debug
28 - use AvailableCoinsType instead of string in walletmodel
29 - New inventory system for IX messaging, providing super fast/low bandwidth IX communication
