191 // Require that user allocate at least 945MB for block & undo files (blk???.dat and rev???.dat)
192 // At 2MB per block, 288 blocks = 576MB.
193 // Add 15% for Undo data = 662MB
194 // Add 20% for Orphan block rate = 794MB
195 // We want the low water mark after pruning to be at least 794 MB and since we prune in
196 // full block file chunks, we need the high water mark which triggers the prune to be
197 // one 128MB block file + added 15% undo data = 147MB greater for a total of 941MB
198 // Setting the target to > than 945MB will make it likely we can respect the target.
199 static const uint64_t MIN_DISK_SPACE_FOR_BLOCK_FILES = 945 * 1024 * 1024;
